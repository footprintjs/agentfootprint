# agentfootprint docs

A Next.js 16 + [Fumadocs](https://fumadocs.dev) (React-native, AI-native) documentation
site for **agentfootprint** — the published docs at
https://footprintjs.github.io/agentfootprint/. (Migrated from a former Astro/Starlight
site, now removed.)

## Why Fumadocs (vs the Starlight site)

- **React-native** — the interactive "Run → runtime flowchart next to the code" feature
  (porting `explainable-ui` + the `footprint-playground` runner) is first-class here,
  not an island bolted onto a non-React framework.
- **AI-native** — `llms.txt` / `llms-full.txt`, per-page raw Markdown, and MCP are
  native rather than plugin-bolted (on-brand: our whole thesis is machine-legible systems).
- **Anti-drift preserved** — examples-as-tests + build-time code-from-source + Twoslash
  (compile-checked code blocks) carry over from the Starlight discipline; Twoslash also
  closes the gap where hand-written inline snippets could rot.

## Develop

```bash
npm install      # also runs `fumadocs-mdx` (codegen → .source/)
npm run dev      # http://localhost:3000 → /docs
npm run build    # static/server build
```

## Layout

| Path | What |
|---|---|
| `content/docs/**.mdx` | the documentation content |
| `source.config.ts` | Fumadocs content config (frontmatter schema, dir) |
| `lib/source.ts` | the content-source loader (`/docs` base) |
| `app/docs/[[...slug]]/page.tsx` | the docs route |
| `app/api/search/route.ts` | built-in Orama search endpoint |
| `.source/` | generated by `fumadocs-mdx` — **not committed** |

## Auto-generated API reference (anti-stale infra)

The full API reference under `content/docs/api/**` is **generated from TypeScript
source** by [`../scripts/gen-fumadocs-api.mjs`](../scripts/gen-fumadocs-api.mjs)
(TypeDoc + `typedoc-plugin-markdown`, config in
[`../typedoc.docs-next.json`](../typedoc.docs-next.json)). The script:

1. **cleans** `content/docs/api/` then regenerates — a removed/renamed export leaves
   **no stale page** behind;
2. post-processes each page to be Fumadocs-ready (adds `title` frontmatter, rewrites
   internal links, writes sidebar `meta.json`).

It runs as **`predev` / `prebuild`**, so dev and production builds always reflect the
current source. The generated `.md` files **are committed** (reviewable diffs + the
site builds without re-running TypeDoc).

```bash
npm run gen:api      # regenerate the API reference now
npm run check:api    # CI guard: regenerate + `git diff --exit-code` (fails if stale)
```

Wire `npm run check:api` into CI so a PR that changes the public API but forgets to
regenerate docs fails the build — the no-stale-docs guarantee.
