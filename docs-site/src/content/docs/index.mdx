## The one thing that's different

Every other agent framework ships agents you debug with **logs**. `agentfootprint` ships agents whose **every decision, tool call, memory write, and context injection is captured as a typed event during one DFS traversal** — no instrumentation, no post-processing.

```ts
import { Agent, defineMemory, MEMORY_TYPES, MEMORY_STRATEGIES, InMemoryStore, mock } from 'agentfootprint';

const agent = Agent.create({ provider: mock({ reply: 'Hi Alice!' }), model: 'mock' })
  .memory(defineMemory({
    id: 'short-term',
    type: MEMORY_TYPES.EPISODIC,
    strategy: { kind: MEMORY_STRATEGIES.WINDOW, size: 10 },
    store: new InMemoryStore(),
  }))
  .build();

const id = { conversationId: 'alice-session' };
await agent.run({ message: 'My name is Alice.', identity: id });
const result = await agent.run({ message: 'What did I just say?', identity: id });

// Inspect what happened
console.log(agent.getNarrative());
agent.on('agentfootprint.context.injected', (e) =>
  console.log(`[${e.payload.source}] landed in ${e.payload.slot} slot`));
```

Every event is typed (47 events × 13 domains). Every memory injection is a `context.injected` event. Every decision is a `decide()` evidence record. Every tool call has start/end events with args + results. The trace is **already there** because the library was built on the flowchart pattern from day one.

## The stack, in one box

```
PRIMITIVES (2)        LLM · Agent (= ReAct)
COMPOSITIONS (3)      Sequence · Parallel · Conditional
PATTERNS (N)          ReAct · Dynamic ReAct · Hierarchy (Swarm) ·
                      Reflexion · Tree-of-Thoughts · Plan-Execute · Map-Reduce
CONTEXT ENGINEERING   defineSkill · defineSteering · defineInstruction · defineFact
                       (one Injection primitive, four typed sugar factories)
MEMORY                4 types × 7 strategies — defineMemory({type, strategy, store})
                       Episodic · Semantic · Narrative · Causal (cross-run replay)
FEATURES              Providers · Recorders · Pause/Resume · Permissions · Streaming
```

Two theses:

1. **Agent = ReAct.** If it doesn't loop-with-tools, it isn't an Agent.
2. **Every named paper is a recipe of primitives + compositions.** Reflexion = `Sequence(Agent, critique-LLM, Agent)`. Tree-of-Thoughts = `Parallel(Agent × N) + rank`. Hierarchy (Swarm) = Agent whose tools are Agents. RAG isn't a primitive; it's context engineering — what you inject into the messages slot.

See the [Key Concepts](/agentfootprint/getting-started/key-concepts/) page for the full taxonomy with worked examples.

## 30-second demo

1. Pick a pattern in [Live Chat](https://footprintjs.github.io/agent-playground/#/live) (works with a mock adapter — no API key needed).
2. Send a message. See the response + the **Behind the Scenes** button.
3. Click **Copy Trace**. Paste it into [Trace Viewer](https://footprintjs.github.io/agent-playground/#/viewer).
4. Drag the time-travel slider. Every step of the agent's reasoning is there.

Zero typing. Zero keys. Everything else in this library follows from this one workflow.

## Install

```sh
npm install agentfootprint footprintjs
```

Peer dep: `footprintjs >= 4.14.0`. Optional: `@anthropic-ai/sdk`, `openai`, `@aws-sdk/client-bedrock-runtime` — depending on the LLM provider you want.
