## The differentiator, concretely

Everyone has tool calling. Everyone has memory. Everyone has multi-agent. What **nobody else has** is the complete debuggability stack:

```ts
// 1. Your agent runs
await agent.run(userMessage);

// 2. Capture the full execution as portable JSON (one line, free)
const trace = exportTrace(agent);

// 3. Later — weeks later, different machine, different team — replay it
<TraceViewer trace={trace} />
```

The nearest competitor is **LangSmith**, which:
- Requires a paid SaaS account
- Requires opt-in instrumentation on every run
- Shows logs, not evidence — no "why this branch?"
- Works only with LangChain stack

`agentfootprint` gives you this for free, offline, with no hosted service, tied to no vendor.

## Where agentfootprint is *not* the right choice

**You're Python-only.** We're TypeScript-first. Port on the roadmap; not soon. Use LangGraph.

**You want a no-code / visual-first experience.** LangGraph Studio is better at that. We're code-first because we believe the code *is* the diagram (see `chart.toMermaid()`).

**You want the largest ecosystem of pre-built tool integrations.** LangChain has 1000+. We have adapters for the top LLM providers and leave custom tools to you (5-line `defineTool`).

**You want role-play multi-agent abstractions out of the box.** CrewAI gives you "roles" as a first-class concept. We give you `Swarm` (specialist dispatch) and `Parallel` (fan-out), which can build CrewAI-style role-play but with less opinion.

**You want zero dependencies.** OpenAI Agents SDK is minimalist. We have more surface area — the trade is introspection.

## Where agentfootprint *is* the right choice

**You need to ship agents to regulated customers.** Compliance requires evidence. Our narrative + commit log is the audit trail by construction.

**Your agents will outlive their authors.** 6 months from now, somebody else debugs a bug from October. Paste the trace into the viewer — they see what the original author would have seen.

**You care about CI cost.** `mock([...])` means your 500-test agent suite runs free in CI. Swap to real adapter with zero code changes for integration runs.

**You want the same framework for LLM call, RAG, agent, and multi-agent.** The taxonomy is 2 primitives (LLM, Agent) + 3 compositions (Sequence, Parallel, Conditional), with RAG / Memory / Skills as context engineering on top. Every named pattern (Reflexion, Tree-of-Thoughts, Hierarchy/Swarm) is a recipe — not a new class. One interface, the whole spectrum.

**You want to debug agents like you debug code.** Time-travel slider. Memory diff per step. "Why did stage 7 get this value?" → causal chain. That's the whole pitch.

## Decision flowchart

```
Are you in a compliance-heavy / regulated domain?
  → Yes: agentfootprint (audit trail is the product)

Do you need Python today?
  → Yes: LangGraph or CrewAI

Is your priority "fastest path from prompt to shipped demo"?
  → Yes: OpenAI Agents SDK (if OpenAI-only) or LangChain (more flexible)

Do you need to debug agents in production, hours / days after they ran?
  → Yes: agentfootprint — this is the core differentiator

Do you want a visual DAG editor?
  → Yes: LangGraph Studio

Otherwise: try agentfootprint. The mock adapter + sample trace in the
playground let you evaluate the full debuggability stack in 10 minutes
with no install, no keys, no signup.
```
